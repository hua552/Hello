# git使用
## git init 初始化仓库
建立一个目录，然后在该目录内进行指令init
## git status 仓库状态查看
显示在当前仓库的处在的分支，以及可以提交的内容
## git add向暂存区添加文件
这里是创建一个文件，但是不会直接添加到仓库中，而是进入暂存区，状态查看是在changes to be commited
## git commit 保存仓库的历史记录
将当前暂存区的文件实际保存到仓库的历史记录，从而方便复原
1.git commit -m "F1",F1称做提交信息，是对提交概述
2.不加-m，就会进入编辑器，第一行简述提交的更改内容，第二行空，第三行以后详细信息
（提交全空自动终止提交）
### git commit --amend,修改提交信息，会启动类似的编辑器
## git log查看提交日志
--pretty=short可以只显示提交信息的第一行
加后缀文件，目录名就只有指定的日志

-p 显示文件前后差别（注意要加文件名）
### git reflog是查看仓库的操作日志，在日志中找出回溯历史前的状态，因为log正常只查看当前节点作为终点的日志
## git diff，查看当前工作树和暂存区的差别
git diff HEAD 表示指向当前分支最新一次提交的指针，本次即将提交和上次提交区别

## fork 分支操作
### git branch 查看当前分支，以及分支一览表
### git checkout -b 新分支名  创建、切换分支
以当前的master分支为基础创建新的分支，不加就要先branch 新分支名再输入
### 这样新分支下提交就叫做培育分支

### （直接只checkout就是切换分支），git checkout - 是切换到上一个分支
### 特性分支
集中实现单一特性的分支，除此之外不进行任何作业的分支，同时留下一个随时可以发布的稳定分支
### 主干分支
一般master，特性分支的原点，也是合并的终点，都是完全代码
### git merge——合并分支 
先要 换到想要合并到的分支，然后git merge --no-ff 被合并的分支（参数是创建合并的提交commit的为了方便记录）
### git log --graph——以图表的形式查看分支

## git reset——回溯历史版本
** 注意hash值只用输入4位以上就能执行（前几位）**
git reset --hard 目标时间点的哈希值（**git log（relog，如果不是当前终点的状态）里面得到的对应值**）

![git1](git1.jpg)
这里图的意思，中间表示主干分支，发生了几次合并，其中A合并，然后B也合并

## git rebase -i  HEAD~n,n表示HEAD最新分支往前n-1个，包含自己,压缩历史
合并特性分支之前，如果已提交内容有错，可以提交一个修改，然后将其包含到前一个提交之中，压缩成一个历史记录
使用后会打开编辑器然后显示部分提交日志
将其中pick改为fixup
